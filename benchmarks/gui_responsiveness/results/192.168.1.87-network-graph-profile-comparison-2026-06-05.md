## Notes

- The direct payload reduction is stable: the graph subview now polls `profile=network-graph`, which keeps graph nodes, links, and top-level packet routing fields while dropping port counts, node packet trends, and raw packet bodies.
- One-iteration GUI timings remain noisy on the target and are often quantized around the 3s poll cadence. The first after-run also showed unrelated spikes in settings/chat/console, so timing changes should be interpreted cautiously.
- First graph scene render remains the largest frontend opportunity. The remaining cost appears to be SVG layout/paint work rather than API payload size alone.
