# GUI Responsiveness: Status/Console State Profiles

Target: `http://192.168.1.87:8877/`

Compared runs:

- Before: `192.168.1.87-after-graph-layout-cache-full-2026-06-05.json`
- After: `192.168.1.87-after-status-console-profiles-full-2026-06-05.json`

## Change

Added two lightweight state profiles:

- `profile=status`: summary + slim nodes, no chat rows, packets, edges, port counts, or packet trends.
- `profile=console`: summary + slim nodes + slim recent packets, no chat rows, route edges, port counts, or packet trends.

Polling now uses:

- `settings` -> `status`
- `console` -> `status` normally, `console` when live packet mode is enabled

## Notes

- The payload reduction is deterministic and confirmed by the benchmark API probes.
- The settings view now reports `pollProfile: "status"` in the saved benchmark sample.
- Console normally reports `pollProfile: "status"` unless live packet mode is active. The `console` profile is available for live mode and keeps the recent packet feed.
- `poll:console` regressed in this one run despite smaller payloads, while `switch:console` improved. Rerun with more iterations before treating the console timing as stable.
