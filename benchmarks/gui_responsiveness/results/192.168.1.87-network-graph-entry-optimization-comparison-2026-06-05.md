## Findings

- The original graph entry path rendered the state payload edge set first, then redrew after `/api/history/links` returned. That produced two full SVG replacements and extra data-sync passes.
- Gating graph rendering to the `network-graph` poll profile removed the duplicate profile-driven patch passes.
- Waiting for history edges before drawing the graph removed the stale 633-edge replacement and left a cheap `history-loading` pass plus one real 372-edge replacement.
- Narrowing the scene data signature stopped ordinary backend state-counter churn from forcing an O(edges + nodes) SVG data sync on graph polls.

Overall sample timings remain noisy because unrelated views can spike during a one-iteration run. The graph-specific render action counts are the stronger signal here: entry dropped from 8 render runs with 2 full replacements to 2 render runs with 1 full replacement, and graph poll now skips SVG data sync.
