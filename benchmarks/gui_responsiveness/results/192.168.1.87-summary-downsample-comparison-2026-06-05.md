## Caveat

The benchmark's requested `history` view currently normalizes to `chat` because the public v0 rail exposes Chat/Network/Console/Games/BBS/Files/Settings, not History. This run still measures the summary endpoint as an API payload, but it does not provide a direct history-screen switch timing.

## Next Optimization Targets

- Network graph: cache/reuse the expensive node-map/layout work when the graph source signature has not changed.
- State payloads: add a lighter profile for settings/console views so they do not pull full chat, edge, and history-cap data.
- Chat polling: reduce or paginate `recent_chat` and skip roster/feed reconciliation when message keys and visible roster inputs are unchanged.
- Benchmark fidelity: add a history-panel-specific benchmark path or stop labeling normalized-away views as distinct samples.
